corpkit.interrogator module¶

corpkit.interrogator.interrogator(path, search, query='any', show='words', exclude=False, case_sensitive=False, lemmatise=False, titlefilter=False, lemmatag=False, spelling=False, phrases=False, dep_type='collapsed-ccprocessed-dependencies', quicksave=False, printstatus=True, root=False, df1_always_df=False, just_speakers=False, excludemode='any', searchmode='all', **kwargs)[source]¶

Interrogate a corpus of texts for a lexicogrammatical phenomenon

Parameters:
  • path (str – corpus path; list of strings – list of paths) – Path to a corpus
  • search (str, or, for dependencies, a dict like {‘w’: ‘help’, ‘p’: r’^V’}) – What query should be matching - t/tregex - w/word - l/lemma - f/function - g/governor - d/dependent - p/pos - i/index - n/ngrams
  • searchmode (str (‘any’/’all’)) – Return results matching any/all criteria
  • exclude (dict – {‘l’: ‘be’}) – The inverse of search, removing results from search
  • excludemode (str (‘any’/’all’)) – Exclude results matching any/all criteria
  • query (str – regex/Tregex pattern; dict – {name: pattern}; list – word list to match) – A search query for the interrogation
  • show (list of strings) – What to output. If multiple strings are passed, results will be colon-separated, in order - t/tree - w/word - l/lemma - g/governor - d/dependent - f/function - p/pos - i/index - a/distance from root
  • lemmatise (bool) – Do lemmatisation on results
  • lemmatag (False/’n’/’v’/’a’/’r’) – Explicitly pass a pos to lemmatiser (generally when data is unparsed)
  • titlefilter (bool) – Strip ‘mr, ‘the’, ‘dr.’ etc. from multiword results (turns ‘phrases’ on)
  • spelling (False/’US’/’UK’) – Convert all to U.S. or U.K. English
  • phrases (bool) – Use if your expected results are multiword (e.g. searching for NP, with show as ‘w’), and thus need tokenising
  • dep_type (str – ‘basic-dependencies’/’a’, ‘collapsed-dependencies’/’b’, ‘collapsed-ccprocessed-dependencies’/’c’) – The kind of Stanford CoreNLP dependency parses you want to use:
  • quicksave (str) – Save result as pickle to saved_interrogations/quicksave on completion
  • gramsize (int) – size of ngrams (default 2)
  • split_contractions (bool) – make “don’t” et al into two tokens
  • num_proc (int) – how many parallel processes to run – default is the number of cores
Returns:

A named tuple, with .query, .results, .totals attributes. If multiprocessing is invoked, result may be a dict containing corpus names, queries or speakers as keys.

corpkit.editor module¶

corpkit.editor.editor(dataframe1, operation=None, dataframe2=False, sort_by=False, keep_stats=False, keep_top=False, just_totals=False, threshold='medium', just_entries=False, skip_entries=False, merge_entries=False, newname='combine', multiple_merge=False, just_subcorpora=False, skip_subcorpora=False, span_subcorpora=False, merge_subcorpora=False, new_subcorpus_name=False, replace_names=False, projection=False, remove_above_p=False, p=0.05, revert_year=True, print_info=True, spelling=False, selfdrop=True, calc_all=True, **kwargs)[source]¶

Edit results of interrogations, do keywording, sort, etc.

just/skip_entries and just/skip_subcorpora can take a few different kinds of input:

  • str: treated as regular expression to match
  • list:
    • of integers: indices to match
    • of strings: entries/subcorpora to match

merge_entries and merge_subcorpora, however, are best entered as dicts:

{newname: criteria, newname2: criteria2}`

where criteria is a string, list, etc.

Parameters:
  • dataframe1 (pandas.core.frame.DataFrame) – Results to edit
  • operation (str) –

    Kind of maths to do on inputted lists:

    ‘+’, ‘-‘, ‘/’, ‘*’, ‘%’: self explanatory ‘k’: log likelihood (keywords) ‘a’: get distance metric (for use with interrogator ‘a’ option) ‘d’: get percent difference (alternative approach to keywording)

  • dataframe2 (pandas.core.series.Series/pandas.core.frame.DataFrame/dict/’self’) –

    List of results or totals.

    If list of results, for each entry in dataframe 1, locate entry with same name in dataframe 2, and do maths there if ‘self’, do all merging/keeping operations, then use edited dataframe1 as dataframe2

  • sort_by (str) –

    Calculate slope, stderr, r, p values, then sort by:

    increase: highest to lowest slope value decrease: lowest to highest slope value turbulent: most change in y axis values static: least change in y axis values total/most: largest number first infreq/least: smallest number first name: alphabetically

  • keep_stats (bool) – Keep/drop stats values from dataframe after sorting
  • keep_top (int) – After sorting, remove all but the top keep_top results
  • just_totals (bool) – Sum each column and work with sums
  • threshold (int/bool) –

    When using results list as dataframe 2, drop values occurring fewer than n times. If not keywording, you can use:

    'high': dataframe2 total / 2500 'medium': dataframe2 total / 5000 'low': dataframe2 total / 10000

    Note: if keywording, there are smaller default thresholds

  • just_entries (see above) – Keep matching entries
  • skip_entries (see above) – Skip matching entries
  • merge_entries (see above) – Merge matching entries
  • newname (str/’combine’) – New name for merged entries
  • just_subcorpora (see above) – Keep matching subcorpora
  • skip_subcorpora (see above) – Skip matching subcorpora
  • span_subcorpora (tuple – (int, int2)) – If subcorpora are numerically named, span all from int to int2, inclusive
  • merge_subcorpora (see above) – Merge matching subcorpora
  • new_subcorpus_name (str/'combine') – Name for merged subcorpora
  • replace_names (dict – {criteria: replacement_text}; str – a regex to delete from names) – Edit result names and then merge duplicate names.
  • projection (tuple – (subcorpus_name, n)) – a to multiply results in subcorpus by n
  • remove_above_p (bool) – Delete any result over p
  • p (float) – set the p value
  • revert_year (bool) – when doing linear regression on years, turn annual subcorpora into 1, 2 ...
  • print_info (bool) – Print stuff to console showing what’s being edited
  • spelling (str – 'US'/'UK') – Convert/normalise spelling:
  • selfdrop (bool) – When keywording, try to remove target corpus from reference corpus
  • calc_all (bool) – When keywording, calculate words that appear in either corpus
Returns:

Edited interrogation
